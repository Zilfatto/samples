My project

Why did I choose this subject?

Firstly, I am not so good in English for writing perfect articles and secondly, I made it for showing my developer's skills, but not copywriter's ones. So I was finding subjects where I did not need to generate most of the text content by myself and could just copy and paste it. (and of course sources that were available for non-commercial using)

What I did not do in it or did wrong

  • I started creating first pages for “desktop first”, then realised that it was not the best decision, but it was quite late for changing it because it would take quite a lot of time. So I left it as it was.
  • I did not find any ways for caching only images in Flask, so they are always loading.
  • I used a jpeg format for images, but there are next-gen ones for better compression. I needed to convert copies of that images to WebP format and add links to them to a picture HTML elements, but it also would take quite a lot of time.
  • I did not reduce the size of all files using popular bundler - Webpack, because I wanted to let all visitiors see the source code not only going to the GitHub, but also directly from the current page
  • I did not provide support of modern JavaScript for all browsers, because at the moment of bulding this project I was not so familliar with the technology like Babel. May be in the future, when I have enough free time, I will fix that.

My web app is not for the production, so I only write about these details here in order to say that I have an understanding of it.

All HTML and CSS in my project successfully passed the validation on Markup Validation Service and support all modern browsers starting from IE10. But there is one subtle - I implemented loading some images after the body has loaded or a user has scolled curtain place on the page using JavaScript.head-img To achieve this I needed to remove "href" attributes from <img> elements
